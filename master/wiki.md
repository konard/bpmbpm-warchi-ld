# ДОКУМЕНТАЦИЯ warchi

> Источник: [warchi.ru/docs/overview](https://warchi.ru/docs/overview)  
> Проект: [warchi-ld](https://github.com/bpmbpm/warchi-ld)

---

## Обзор

**warchi** (wArchi) — браузерный архитектурный редактор для моделирования бизнес-процессов и архитектуры предприятия. Инструмент позволяет строить диаграммы в нотации VAD (Value Added Diagram / Цепочка добавленной стоимости) и ArchiMate-совместимых нотациях.

**warchi-ld** — это Linked Data версия warchi (по аналогии JSON vs JSON-LD): все данные хранятся в формате RDF/TriG, что позволяет использовать semantic reasoning, SPARQL-запросы и интеграцию с другими Linked Data системами.

## ДОКУМЕНТАЦИЯ

### 1. Основные концепции

#### 1.1 Нотация VAD (Value Added Diagram)

VAD (Цепочка добавленной стоимости) — нотация для описания верхнеуровневых бизнес-процессов. Ключевые элементы:

- **Процесс (TypeProcess)** — основной объект, представляющий бизнес-процесс или функцию
- **Исполнитель (TypeExecutor)** — роль, подразделение или должность, выполняющая процесс
- **Группа исполнителей (ExecutorGroup)** — объединение исполнителей для конкретного процесса
- **Связь «следующий» (hasNext)** — переход между процессами в цепочке
- **Связь «исполнитель» (hasExecutor)** — кто выполняет данный процесс

#### 1.2 Иерархия TriG

Данные хранятся в именованных графах (TriG):

| Граф | Тип | Описание |
|------|-----|----------|
| `vad:root` | TechTree | Корневой узел всего дерева |
| `vad:ptree` | ObjectTree | Дерево процессов — метаданные всех процессов |
| `vad:rtree` | ObjectTree | Дерево исполнителей — метаданные всех исполнителей |
| `vad:t_*` | VADProcessDia | Схема конкретного процесса (диаграмма) |

#### 1.3 Подтипы процессов (VAD Node Styles)

Каждый процесс на диаграмме получает подтип, определяющий его визуальное отображение:

| Подтип | Описание | Цвет |
|--------|----------|------|
| `DetailedChild` | Детализированный подпроцесс (имеет схему, является дочерним) | Синий |
| `DetailedExternal` | Детализированный внешний процесс | Голубой |
| `notDetailedChild` | Не детализированный подпроцесс | Зелёный |
| `notDetailedExternal` | Не детализированный внешний процесс | Серый |
| `NotDefinedType` | Процесс с неопределённым родителем (`pNotDefined`) | Жёлтый |

### 2. Интерфейс программы

#### 2.1 Строка меню

- **Файл** — управление проектами: открыть, сохранить, импорт/экспорт
- **Правка** — редактирование данных
- **Вид** — управление отображением панелей
- **Нотация** — выбор нотации (VAD, Custom)
- **Инструменты** — Virtual TriG, Reasoning, Валидация RDF
- **Справка** — документация и сведения о программе

#### 2.2 Линейка инструментов (Toolbar)

Основные кнопки быстрого доступа:
- Новый проект, Открыть файл, Сохранить
- Обновить диаграмму, Экспорт SVG/PNG
- Лог (окно отладки)
- Фильтр TriG — выбор набора данных для отображения

#### 2.3 Боковая панель (Навигатор)

- **Вкладка «Дерево»** — иерархическое дерево процессов (ptree) и исполнителей (rtree)
- **Вкладка «Свойства»** — свойства выбранного элемента

#### 2.4 Рабочая область (панели)

Основные панели (сворачиваемые):

1. **Загрузить пример данных** — быстрая загрузка тестовых файлов из папки `/dia`
2. **RDF данные (quadstore)** — просмотр и управление данными в хранилище
3. **Smart Design_** (Диаграмма как код) — вторичный блок для создания схем через SPARQL
4. **Диаграмма** — область визуализации VAD-диаграммы
5. **Легенда** — расшифровка цветов и стилей
6. **Префиксы и TriG дерево** — информация о пространствах имён
7. **SPARQL запросы** — выполнение SPARQL-запросов к хранилищу

### 3. Работа с файлами

#### 3.1 Поддерживаемые форматы

| Формат | Расширение | Описание |
|--------|-----------|----------|
| TriG | `.trig`, `.ttl` | Основной формат хранения (RDF Named Graphs) |
| Turtle | `.ttl` | RDF Turtle (без именованных графов) |
| warchi | `.warchi` | Собственный формат warchi.ru (JSON) |

#### 3.2 Открытие файла

**Способ 1 — Выбор примера:**
1. В панели «Загрузить пример данных» выберите файл из списка
2. Файл загрузится автоматически и диаграмма визуализируется

**Способ 2 — Открыть через кнопку:**
1. В панели «RDF данные» нажмите «Загрузить» или используйте меню «Файл → Открыть TTL/TriG…»
2. Выберите файл в диалоге
3. Нажмите «Визуализировать» в панели «Диаграмма»

**Способ 3 — Импорт warchi:**
1. Меню «Файл → Импорт warchi…»
2. Выберите `.warchi` файл — он автоматически конвертируется в TriG

#### 3.3 Сохранение

- **Сохранить как** (кнопка или меню) — скачивает текущий quadstore как `.ttl` файл
- **Экспорт SVG/PNG** — экспорт диаграммы как изображение
- **Экспорт в warchi** — конвертация TriG в формат `.warchi`

### 4. Онтология VAD

Онтология определяет словарь данных проекта. Файлы:

- `ontology/vad-basic-ontology.trig` — базовая онтология VAD
- `ontology/vad-basic-ontology_tech_Appendix.trig` — технологические классы

#### 4.1 Классы онтологии

| Класс | Описание |
|-------|----------|
| `vad:TypeProcess` | Бизнес-процесс |
| `vad:TypeExecutor` | Исполнитель |
| `vad:ExecutorGroup` | Группа исполнителей процесса |
| `vad:VADProcessDia` | Схема процесса (именованный граф) |
| `vad:ObjectTree` | Дерево объектов (ptree, rtree) |
| `vad:TechTree` | Корневой технический узел |
| `vad:Virtual` | Виртуальные вычисляемые данные |

#### 4.2 Свойства онтологии

| Свойство | Домен | Диапазон | Описание |
|----------|-------|----------|----------|
| `vad:hasParentObj` | любой | любой | Родительский объект в иерархии |
| `vad:hasTrig` | Process | VADProcessDia | Ссылка на детализирующую схему |
| `vad:hasNext` | Process | Process | Следующий процесс в цепочке |
| `vad:hasExecutor` | Process | ExecutorGroup | Исполнитель процесса |
| `vad:isSubprocessTrig` | Process | VADProcessDia | Вхождение процесса в схему |
| `vad:includes` | ExecutorGroup | Executor | Состав группы исполнителей |
| `vad:processSubtype` | Process | — | Подтип (вычисляется автоматически) |
| `vad:x` | Process | xsd:integer | Координата X фигуры на диаграмме |
| `vad:y` | Process | xsd:integer | Координата Y фигуры на диаграмме |

### 5. Редактор диаграмм

#### 5.1 Классический редактор (drag-and-drop)

Классический редактор позволяет создавать диаграммы путём перетаскивания фигур из палитры на холст:

1. В палитре слева выберите тип фигуры (процесс, группа исполнителей)
2. Перетащите фигуру на холст
3. Двойной клик — редактирование подписи
4. При перемещении фигуры координаты `vad:x`, `vad:y` обновляются в RDF-данных

#### 5.2 Smart Design (Диаграмма как код)

Smart Design — вторичный блок для создания схем через интерфейс на основе SPARQL:

- **✚ Концепт** — создать новый концепт процесса в ptree
- **✚ Индивид** — создать индивид в схеме (VADProcessDia)  
- **✚ Схема_** — создать новый граф VADProcessDia

> Примечание: При создании схем через Smart Design координаты `vad:x`, `vad:y` подставляются автоматически.

### 7. Быстрый старт

1. Откройте `ver1/index.html` в браузере (Chrome, Firefox, Edge)
2. В панели «Загрузить пример данных» выберите `Trig_VADv8.ttl`
3. Диаграмма откроется автоматически
4. Используйте дерево в боковой панели для навигации по процессам
5. Нажмите «Обновить» для перерисовки диаграммы
