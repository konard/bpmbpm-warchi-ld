# ДОКУМЕНТАЦИЯ warchi

> Источник: [warchi.ru](https://warchi.ru)

---

## Обзор

**warchi** (wArchi) — браузерный архитектурный редактор для моделирования бизнес-процессов и архитектуры предприятия. Инструмент позволяет строить диаграммы в нотации VAD (Value Added Diagram / Цепочка добавленной стоимости) и ArchiMate-совместимых нотациях.

---

## 1. Нотация VAD (Value Added Diagram)

VAD (Цепочка добавленной стоимости) — нотация для описания верхнеуровневых бизнес-процессов. Ключевые элементы:

- **Процесс** — основной объект, представляющий бизнес-процесс или функцию. Левая сторона элемента прямая, правая — стрелка.
- **Исполнитель** — роль, подразделение или должность, выполняющая процесс
- **Группа исполнителей** — объединение исполнителей для конкретного процесса
- **Связь «следующий»** — переход между процессами в цепочке
- **Связь «исполнитель»** — кто выполняет данный процесс

### 1.1 Подтипы процессов

| Подтип | Описание | Цвет |
|--------|----------|------|
| Детализированный подпроцесс | Имеет схему, является дочерним | Синий |
| Детализированный внешний | Детализированный внешний процесс | Голубой |
| Не детализированный подпроцесс | Не имеет схемы, дочерний | Зелёный |
| Не детализированный внешний | Не детализированный внешний | Светло-зелёный |
| Неопределённый тип | Процесс с неопределённым родителем | Серый |

## 2. Интерфейс warchi

### 2.1 Строка меню

- **Файл** — управление проектами: открыть, сохранить, экспорт
- **Правка** — редактирование данных
- **Вид** — управление отображением панелей
- **Нотация** — выбор нотации (VAD)
- **Инструменты** — дополнительные инструменты
- **Справка** — документация и сведения о программе

### 2.2 Линейка инструментов (Toolbar)

Основные кнопки быстрого доступа:
- Новый проект, Открыть файл, Сохранить
- Обновить диаграмму, Экспорт SVG/PNG
- Лог — журнал событий и отладки
- Фильтр TriG — выбор набора данных для отображения

### 2.3 Боковая панель (Навигатор)

- **Вкладка «Дерево»** — иерархическое дерево процессов и исполнителей
- **Вкладка «Свойства»** — свойства выбранного элемента
- **Вкладка «Трафарет»** — палитра элементов VAD-нотации для добавления на диаграмму

### 2.4 Трафарет VAD

Трафарет содержит все элементы из онтологии VAD:

**Узлы:**
- Процесс (детализированный дочерний) — синий, левая сторона прямая
- Процесс (детализированный внешний) — голубой, левая сторона прямая
- Процесс (не детализированный дочерний) — зелёный, левая сторона прямая
- Процесс (не детализированный внешний) — светло-зелёный, левая сторона прямая
- Процесс (неопределённый тип) — серый, левая сторона прямая
- Группа исполнителей — эллипс, жёлтый
- Исполнитель — эллипс, фиолетовый

**Связи:**
- Следующий — зелёная стрелка (сплошная)
- Исполнитель — синяя пунктирная линия
- Включает — фиолетовая точечная линия
- Родитель — серая пунктирная линия
- Тип — фиолетовая пунктирная стрелка

### 2.5 Рабочая область (панели)

Основные панели (сворачиваемые):

1. **Загрузить пример данных** — быстрая загрузка тестовых файлов
2. **Диаграмма** — область визуализации VAD-диаграммы
3. **Легенда** — расшифровка цветов и стилей
4. **Префиксы и TriG дерево** — информация о пространствах имён
5. **SPARQL запросы** — выполнение SPARQL-запросов
6. **Матрица связей компонентов** — матрица переходов между процессами (vad:hasNext)
7. **Лог** — журнал событий (самое нижнее окно)

## 3. Работа с файлами

### 3.1 Поддерживаемые форматы

| Формат | Расширение | Описание |
|--------|-----------|----------|
| TriG | `.trig`, `.ttl` | Основной формат хранения (RDF Named Graphs) |
| Turtle | `.ttl` | RDF Turtle (без именованных графов) |

### 3.2 Открытие файла

**Способ 1 — Выбор примера:**
1. В панели «Загрузить пример данных» выберите файл из списка
2. Файл загрузится автоматически и диаграмма визуализируется

**Способ 2 — Открыть через кнопку:**
1. Используйте меню «Файл → Открыть TTL/TriG…» или кнопку на тулбаре
2. Выберите файл в диалоге
3. Диаграмма визуализируется автоматически

### 3.3 Сохранение

- **Сохранить как** (кнопка или меню) — скачивает текущие данные как файл
- **Экспорт SVG/PNG** — экспорт диаграммы как изображение

---

## 4. Матрица связей компонентов

Матрица отображает все связи вида «следующий» (hasNext) между процессами.

- Строки — исходные процессы
- Столбцы — целевые процессы
- Ячейка ✓ — есть связь hasNext от строки к столбцу
- Матрица обновляется автоматически при загрузке данных

---

## 5. Лог (журнал событий)

Самое нижнее окно интерфейса. Отображает все действия пользователя и системные сообщения в хронологическом порядке с временными метками.
