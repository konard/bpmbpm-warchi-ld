# improvements_v1.md — Предложения по улучшению warchi-ld

## 1. Хранение данных

**I-1. LocalStorage / IndexedDB для сохранения сессии**
Сохранять quadstore в IndexedDB при каждом изменении. Восстанавливать при загрузке страницы.
Приоритет: высокий.

**I-2. dia/index.json для динамического списка примеров**
Генерировать файл `dia/index.json` при коммите (GitHub Action) со списком всех файлов в `dia/`. Заменить статический список в `scanDiaFolder()`.

## 2. Конвертация форматов

**I-3. Полноценный импорт/экспорт warchi XML**
Получить спецификацию XML-формата warchi.ru и реализовать полную двустороннюю конвертацию.

**I-4. Импорт/экспорт ArchiMate XML (Archi tool)**
Добавить конвертер из/в формат Archi (.archimate), чтобы проект был совместим с экосистемой ArchiMate.

## 3. Пользовательский интерфейс

**I-5. Drag-and-drop в диаграмме**
Реализовать перетаскивание узлов на SVG-диаграмме для изменения порядка процессов в цепочке.

**I-6. Inline редактирование меток**
Двойной клик на узле диаграммы — редактирование метки прямо на схеме.

**I-7. Поиск по дереву**
Уже частично реализован в `5_publisher_treeview_search.js`. Добавить глобальный поиск по всем объектам.

**I-8. История изменений (undo/redo)**
Стек отмены/повтора операций Smart Design.

**I-9. Сохранение состояния панелей в localStorage**
Запоминать, какие панели свёрнуты/развёрнуты.

## 4. Нотации

**I-10. Custom нотации GUI**
Создать визуальный редактор для добавления custom нотаций: новые классы, свойства, визуальные стили.

**I-11. Поддержка нескольких нотаций**
Добавить возможность работы с несколькими нотациями одновременно (напр. VAD + Custom). Переключение через меню «Нотация».

## 5. Производительность

**I-12. Асинхронная визуализация**
Перенести рендеринг Graphviz в Web Worker, чтобы не блокировать UI.

**I-13. Incremental SPARQL**
Кэшировать результаты SPARQL-запросов и обновлять только при изменении затронутых триплетов.

## 6. Интеграция

**I-14. GitHub Pages автодеплой**
Добавить GitHub Actions workflow для автоматического деплоя на GitHub Pages при пуше в main.

**I-15. URL-параметры для загрузки файла**
Поддержка параметра `?file=URL` для загрузки TriG из произвольного URL при открытии страницы.

## 7. Документация и тестирование

**I-16. Автотесты**
Добавить unit-тесты для функций конвертации и SPARQL-запросов (через Vitest или Jest).

**I-17. Примеры для всех нотаций**
Создать несколько примеров TTL для разных бизнес-доменов (организационная структура, IT-архитектура и т.д.).
