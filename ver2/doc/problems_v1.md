# problems_v1.md — Проблемы и ограничения warchi-ld ver1

## 1. Проблемы при изучении warchi.ru

Доступ к сервису warchi.ru не выполнялся (согласно условиям задачи — реализация на базе rdf-grapher/ver9d). Ниже перечислены предполагаемые и фактические проблемы на основе анализа кода.

## 2. Технические ограничения

### 2.1 CDN-зависимости

Библиотеки N3.js, Comunica и Viz.js загружаются с CDN (unpkg.com). При работе без интернет-соединения потребуется:
- Скопировать библиотеки локально в папку `lib/`
- Обновить пути в `index.html`

### 2.2 Comunica SPARQL — ограничения браузерной версии

- Не все SPARQL 1.1 функции поддерживаются в `@comunica/query-sparql-rdfjs`
- UPDATE операции (INSERT/DELETE) выполняются через N3.Store напрямую, не через SPARQL UPDATE движок
- Reasoning (RDFS/OWL) работает через N3-правила, не через полный OWL-reasoner

### 2.3 Сканирование папки /dia

GitHub Pages и браузеры не позволяют автоматически сканировать содержимое папки. Список файлов в `warchi_ld_ui.js::scanDiaFolder()` задаётся статически. При добавлении новых файлов в `dia/` нужно обновить список.

**Решение в ver2**: использовать `dia/index.json` — список файлов генерируется при сборке.

### 2.4 Масштабирование диаграмм

Механизм zoom реализован через CSS `transform: scale()`. Для больших диаграмм может потребоваться более сложное решение (например, SVG viewBox manipulation или библиотека пансорования).

### 2.5 Конвертация форматов warchi XML

Поддержка XML-формата warchi реализована как базовая заглушка через `DOMParser`. Полная поддержка зависит от точной спецификации XML-схемы warchi.ru (не была доступна).

### 2.6 Custom нотация

Механизм custom нотации реализован на уровне загрузки дополнительных онтологий из папки `ontology/`. Полноценный GUI для добавления custom нотаций (аналог warchi.ru) требует доработки.

### 2.7 Сохранение состояния сессии

Текущая версия не сохраняет состояние между сессиями (нет localStorage/IndexedDB). При перезагрузке страницы загруженные данные теряются.

## 3. Ограничения нотации

- Реализована только нотация VAD. Нотации ArchiMate, C4, BPMN, UML не реализованы согласно требованию issue #1 п.1.3.
- Визуальные стили VAD-элементов заимствованы из rdf-grapher/ver9d без изменений.

## 4. Проблемы производительности

- Для больших quadstore (>10000 триплетов) SPARQL-запросы через Comunica могут работать медленно.
- Viz.js/Graphviz работает синхронно и может блокировать UI при больших графах.
