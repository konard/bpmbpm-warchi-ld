# start.md — Инструкция по запуску warchi-ld

## Быстрый старт

### 1. Открытие файла примера Trig_VADv8.ttl

1. Откройте файл `ver1/index.html` в браузере (Chrome, Firefox, Edge).
   - Локально: двойной клик по файлу или `File → Open File` в браузере.
   - На GitHub Pages: перейдите по ссылке проекта.

2. В панели **«Загрузить пример данных (папка /dia)»** выберите из выпадающего списка:
   ```
   Trig_VADv8.ttl
   ```
   
3. Нажмите кнопку **«Визуализировать»** в панели **«Диаграмма»** — VAD-схема отобразится в виде графа.

4. В боковой панели **«Навигатор»** вкладка **«Дерево»** покажет иерархию процессов (ptree) и исполнителей (rtree).

### 2. Открытие файла Trig_VADv8.ttl через кнопку «Загрузить»

1. В панели **«RDF данные (quadstore)»** нажмите кнопку **«Загрузить»**.
2. В диалоге выберите файл `ver1/dia/Trig_VADv8.ttl`.
3. Данные загрузятся в quadstore. Нажмите **«Визуализировать»**.

### 3. Открытие файла в формате warchi

1. В меню **«Файл → Импорт warchi…_»** выберите файл `ver1/dia/Trig_VADv8_warchi.warchi`.
2. Файл будет автоматически конвертирован из warchi JSON в TriG и загружен в quadstore.
3. Нажмите **«Визуализировать»**.

Альтернативный способ: нажмите кнопку **«Загрузить»** в панели quadstore и выберите `.warchi` файл.

### 4. Конвертация TriG → warchi

1. Загрузите TTL/TriG файл в quadstore (кнопка **«Загрузить»**).
2. В меню **«Файл → Экспорт в warchi_»** — файл `diagram.warchi` будет скачан.

### 5. Создание новых схем

Используйте панель **«Smart Design»** или кнопки в линейке инструментов:
- **«Концепт»** — создать новый концепт процесса в ptree.
- **«Индивид»** — создать индивид в схеме (VADProcessDia).
- **«Схема_»** — создать новый граф VADProcessDia.

### 6. Сохранение

- Кнопка **«Сохранить»** (или **«Файл → Сохранить как…»**) скачивает текущий quadstore как `.ttl` файл.
- **«Файл → Экспорт SVG»** / **«Экспорт PNG»** — экспорт диаграммы.

## Описание файлов примеров

| Файл | Формат | Описание |
|------|--------|----------|
| `dia/Trig_VADv8.ttl` | TriG/TTL | Пример VAD с иерархией процессов p1, p2 и исполнителями |
| `dia/Trig_VADv8_warchi.warchi` | warchi JSON | Тот же пример в формате warchi (для тестирования импорта) |

## Онтологии

При загрузке страницы автоматически подключаются:
- `ontology/vad-basic-ontology.trig` — базовая онтология VAD
- `ontology/vad-basic-ontology_tech_Appendix.trig` — технологические классы

Можно скорректировать онтологию в папке `ontology/` и перезагрузить страницу.
